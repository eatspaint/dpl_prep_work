09: Changing Gears

What’s Next?

You’ve learned the basics of HTML/CSS, now it’s time to learn a language that’s a little more powerful. While HTML & CSS are great for building the side of your website that a user sees, a site built in only HTML/CSS can’t really do very much. We need something else to handle accounts, databases, communicating between apps, calculations, and really all of the other cool logic that happens on the ‘back-end’ of websites.

Enter Ruby

We know that we need something to work with all that complicated back end stuff, and there are numerous options that have been around for a long time (PHP, C, C#, Java, etc.) and some newer ones (Python, Elixir, Javascript, etc.). So why Ruby?

We (and thousands of other developers) choose Ruby for our backends for a few reasons:

  1. Ruby is easy to write: While other languages require lots of confusing syntax to write, Ruby writes a lot more like English than other languages.
  2. Ruby is easy to read: Because Ruby is written almost like English, it is much easier to read through and get a general idea of what’s going on without having to dig into a bunch of syntax.
  3. MINSWAN: Ruby was developed by a man that the community lovingly refers to as Matz. Matz developed Ruby because he believed that there should be a language that exists, first-and-foremost, to promote developer happiness. As a developer, this is great. Because of this, there is a general layer of good-feeling coating everything Ruby. Throughout your journey with Ruby, never forget: Matz Is Nice So We Are Nice (MINSWAN).
