Getting Ready to Write Some Code

Before we get started, we should be sure you’re in a good environment to work. There are countless text editors out there, and everyone has their opinion as to why the one they use is the best. While we could go over every editor out there and its pros/cons, we will tell you about the two we recommend and let you try them both to see which you like better.

  • Atom:
    • Cost: Free for life
    • Visual Customization: Since Atom is built in HTML/CSS/Javascript, every part of the app can be customized; Atom also comes with themes and has a built in browser to install more themes
    • Package Manager: Comes with a browser to find and search for popular themes, packages, and add-ons
    • Pros: Atom is built and maintained by Github (a name you will come to know if you don’t already) in web-languages. Because of this, Atom includes git integration, is very customizable, and has a large open-source community supporting it.
    • Cons: Because it was developed in web-languages, Atom can sometimes load very large files a little slower than other editors. Similar to your web browser, if you install too many packages, Atom will take longer to start up than normal.
  • Sublime:
    • Cost: Free trial; $70 to remove pop-up
    • Visual Customization: Sublime comes with themes, some visual aspects can be changed in the JSON configuration
    • Package Manager: User must install package manager through command line, packages must be found online and installed through package menu, no package browser
    • Pros: Sublime was built for performance. Because it was not built in web-languages, it can sometimes handle very large files with a little more speed.
    • Cons: The ‘Purchase Me’ pop-up can be very annoying when trying to develop. There is no browser for themes/packages. Changing settings in Sublime usually means finding the option in a JSON file and changing its value, rather than using a settings menu.

Introducing HTML & CSS

HTML & CSS are the foundations of the web. Without HTML, there would be no content; without CSS, no styling. Shay Howe has written an extremely comprehensive book on the two that has become a respected reference tool for learning the two languages by the entire web-development community. We will begin your ventures in writing code with his introduction of these concepts.

After completing this module, you should understand:

  • What HTML & CSS are used for in the web
  • Some common HTML terms: elements, tags, attributes
  • The general structure and makeup of HTML documents
  • Some basic CSS terminology: selectors, properties, values
  • The types of CSS selectors
  • How to reference CSS in an HTML document
  • How to build a basic web page with HTML and CSS

As you go through this module, be sure to try out the concepts being presented and follow along as best you can in your own text editor. If you are curious about how something works, or how it could be different, try changing it! You’re the developer now! (Plus, you can always undo your changes if something breaks the code)
